# 网页梗化器

一个 Manifest V3 浏览器扩展：一键把当前网页的图片替换成随机表情包，并把文字里的“的”替换成“🉐”、“了”替换成“辣”。

## 功能

- 一键开启/关闭当前网页梗化
- 开关只影响当前标签页，不会自动污染新打开的网站
- 图片替换成内置随机表情包，不依赖外部图片服务
- 文字抽象化：“的” -> “🉐”，“了” -> “辣”
- 支持重新随机图片
- 支持单独开关图片替换和文字替换
- 通过 MutationObserver 自动处理页面新增内容
- 保留原始内容，重复点击幂等，关闭后可恢复

## 本地安装

1. 打开 Chrome 或 Edge 的扩展管理页。
2. 开启“开发者模式”。
3. 选择“加载已解压的扩展程序”。
4. 选择插件目录：`/Users/hongzaitou/Documents/plauging/plugins/web-memefier`。
5. 打开任意普通网页，点击工具栏里的“网页梗化器”图标。

## 开发检查

```bash
npm install
npm run check
npm run test:extension
```

`npm run test:extension` 会用本机 Chrome 跑一张测试页，覆盖重复开启、恢复原图原文、新增 DOM 自动梗化等核心行为。

## 设计说明

扩展不会修改网站服务器上的内容，只修改当前浏览器标签页里的 DOM 显示效果。刷新页面会完全回到网站原样。

幂等策略：

- 内容脚本只初始化一次，避免重复注册监听器。
- 开启状态只保存在当前页面内存里，图片/文字偏好才会跨页面记住。
- 每个被处理的图片节点会记录原始 `src`、`srcset`、`sizes` 或 `poster`。
- 每个被处理的文本节点会记录原始文本，不会破坏链接、加粗等子结构。
- 再次开启时不会覆盖原始备份。
- 关闭或恢复时按备份回滚，并清理扩展标记。

鲁棒性策略：

- 跳过 `script`、`style`、表单输入、代码块、SVG、Canvas、可编辑区域。
- 页面新增图片或文字会自动处理。
- 图片使用扩展内置 SVG 资源，避免跨域、外链失效和隐私问题。
- 在不支持内容脚本的页面会给出提示。
